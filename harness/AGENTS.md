### 技术栈

```
技术栈：Markdown + Shell（一致性检查脚本）
截止时间：无
核心原则：模板即契约 — 模板修改必须同步实例；链路不可跳过
```

### 铁律（绝对不可违反）

1. **模板与实例同步** — 修改 template/ 下任何文件时，必须同步更新 harness/ 下的同名文件
2. **先读后写** — 修改任何文件前，必须先完整读取该文件当前内容
3. **决策可追溯** — 非显而易见的决策必须写入 [决策记录](./DECISIONS.md)
4. **链路不可跳过，禁止预读** — 严格按 → 下一步 逐级执行，不得跳过任何文件，不得自行决定读取顺序
