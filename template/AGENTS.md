### 自进化

用户说"分析"时做：

> 不乱进化，没有时不动 — 渐进式读取 → 节省 token → 提高命中率 → 用户效率和合理性才是终点。

1. 理解全貌 `[必须]` — 按链路读完项目，不跳步
2. 总结经验 `[必须]` — 回溯本次会话的坑、决策、修复
3. 外部学习 `[建议]` — 网络搜索最佳实践/工具/标准；优先官方文档、行业标准（Linux Foundation/AAIF）、高采纳度开源项目；拒绝 AI 生成内容农场；多源交叉验证后补充到项目（如无网络环境可跳过）
4. 检查优化项 `[建议]` — 以外部知识为参照，对照规范逐项审查遗漏和可改进点
5. 审视规范 `[按需]` — 基于优化项发现，审视规范是否需要移除、添加、修改、调整、完善
6. 自清洁 `[必须]` — 整理会话残留，保持项目整洁
   - 检查并提醒：
     - SESSION.md 超过 3 天未更新 → 提醒补充最新进展
     - tasks.md 有已完成未归档任务 → 提醒归档
     - 临时备忘有内容 → 提醒处理
7. 自循环 `[必须]` — 再次验证"分析"结果

最终建议按必要性分级展示给用户：
- `[必须]` — 违反铁律、破坏框架完整性、或上下文必须延续
- `[建议]` — 提升质量或效率，非阻塞
- `[可选]` — 锦上添花

最终等待用户确认操作。
